## 2. Spécifications mathématiques et structure de l'algorithme

La stratégie possède deux niveaux d'adaptabilité basés sur la performance passée (Equity générée par chaque ligne) amortie par un facteur de décroissance (Decay) et amplifiée par un facteur exponentiel (Lambda).

### A. Paramètres de la stratégie (Pydantic / Config)
- La (Lambda, float, défaut 0.02 divisé par 1000 dans le calcul)
- De (Decay %, float, défaut 0.03 divisé par 100 dans le calcul)
- cutout (CutOut Period, int, défaut 0)
- robustness (options ["Narrow", "Medium", "Wide"], défaut "Medium")
- Long_threshold (float, défaut 0.1)
- Short_threshold (float, défaut -0.1)
- ema_len (int, 28), ema_w (float, 1.0)
- hull_len (int, 28), hma_w (float, 1.0) (Note : le script d'origine appelle ta.sma pour "HMA", respectez ou documentez ce comportement)
- wma_len (int, 28), wma_w (float, 1.0)
- dema_len (int, 28), dema_w (float, 1.0)
- lsma_len (int, 28), lsma_w (float, 1.0)
- kama_len (int, 28), kama_w (float, 1.0)

### B. Couche 1 : Génération des 54 Moving Averages
Pour chaque type d'indicateur (EMA, HMA/SMA, WMA, DEMA, LSMA, KAMA) de longueur L :
- Calculer 9 variations de longueurs basées sur la valeur robustness :
  - Narrow : L ± 1, L ± 2, L ± 3, L ± 4
  - Medium : L ± 1, L ± 2, L ± 4, L ± 6
  - Wide : L ± 1, L ± 3, L ± 5, L ± 7
- Cela représente 54 courbes de prix. Celles-ci doivent être pré-calculées sous forme matricielle avec Pandas / Numpy (ou via vectorbt) pour optimiser les performances avant d'entrer dans la boucle Numba.

### C. Couche 2 : Boucle Numba Stateful (@njit(cache=True))
La boucle Numba prend en entrée la matrice de prix et la matrice des 54 indicateurs pré-calculés :
1. Calcul des rendements : R_r = (C_r - C_{r-1}) / C_{r-1}
2. Calcul de l'exponentielle : Pour r >= cutout, e(La) = exp(La * (r - cutout)).
3. Calcul des Signaux Crossover Individuels : Pour chaque indicateur k in [0, 53] :
   - sig_{k, r} = 1 si close_r croise à la hausse ma_{k, r}
   - sig_{k, r} = -1 si close_r croise à la baisse ma_{k, r}
   - Sinon sig_{k, r} = sig_{k, r-1}
4. Calcul des Equities Individuels (Poids Niveau 1) :
   - r_adjust = R_r * e(La)
   - d = 1 - De
   - a = r_adjust si sig_{k, r-1} > 0 else -r_adjust si sig_{k, r-1} < 0 else 0
   - equity_{k, r} = equity_{k, r-1} * d * (1 + a)
   - Borne inférieure : equity_{k, r} = max(equity_{k, r}, 0.25) (valeur initiale 1.0).
5. Aggrégation Niveau 1 (MA Signal) : Pour chacun des 6 types d'indicateurs j in [0, 5] :
   - MA_Signal_{j, r} = round( (sum_{i=0}^8 sig_{9j+i, r} * equity_{9j+i, r}) / (sum_{i=0}^8 equity_{9j+i, r}), 2 )
6. Calcul des Equities Niveau 2 (Poids Niveau 2) :
   - Pour chaque type d'indicateur j, calculer equity_type_{j, r} en utilisant MA_Signal_{j, r} de la même manière que l'equity niveau 1, avec une valeur initiale égale au poids configuré w_j (ema_w, hma_w, etc.).
7. Signal Moyen Final :
   - Average_Signal_r = (sum_{j=0}^5 Cut(MA_Signal_{j, r}) * equity_type_{j, r}) / (sum_{j=0}^5 equity_type_{j, r})
   - Où Cut(x) = 1 si x > 0.49, -1 si x < -0.49, sinon 0.
8. Signaux Directionnels et Filtrage :
   - Mettre à jour direction_r = 1 si Average_Signal_r croise Long_threshold.
   - Mettre à jour direction_r = -1 si Average_Signal_r croise Short_threshold.
   - Émettre un signal d'entrée long si direction_r == 1 et direction_{r-1} == -1.
   - Émettre un signal d'entrée short si direction_r == -1 et direction_{r-1} == 1.
